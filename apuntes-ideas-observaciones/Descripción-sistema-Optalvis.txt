DESCRIPCIÓN: SISTEMA OPTALVIS Y SITUACIÓN ACTUAL
Cuando el cliente confirma el pedido de un lente previo cotización de los cristales y la montura, el personal anota en un sobre los detalles del pedido como ya se ha detallado en el contexto.
La Óptica Optalvis cuenta con un sistema desarrollado en el año 2000. El sistema maneja como base de datos Microsoft Access versión 2000. Para la generación de sus reportes o resúmenes se usa Crystal report. Desconozco el lenguaje de programación usado pero al ver un por la estructura de los archivos y sus tipos parece que fue hecho en su mayoría con Visual Basic.
El sistema está dividido en tres módulos o tres subsistemas, las características de cada módulo es el siguiente:
SISTEMA DE RECETAS: Este sistema se encarga de registrar la prescripción que el doctor prescriptor entrega al paciente o cliente, este sistema tiene opciones para realizar búsquedas en registros ingresados es decir, permite hacer búsquedas en un historial de prescripciones por código. Este sistema tiene un formulario de registro de prescripciones donde se registra fecha de prescripción, nombre y apellidos de cliente, teléfono, medidas de los lentes, esféricas, cilindro, eje, adición, base. También se registran detalles del cristal que el doctor ha prescrito. Por ejemplo. 2 policarbonatos transition grey para lejos. Luego se registra datos del proveedor de los cristales, el modelo de montura que tiene el cliente o en el caso de que cliente quiera un armazón nuevo entonces se registra el modelo del nuevo armazón, luego se registra fecha de entrega, también se registra el nombre del doctor prescriptor. También se registra un código de pedido de los cristales asociado a esa prescripción. y finalmente se registra el número de sobre que sería el código de la orden de trabajo principal asociado a ese cliente.
En su momento fue un módulo que cumplía con lo que se requería, pero los años han pasado y hoy en 2025 se puede confirmar deficiencias muy marcadas tales como: la sección de búsquedas no funciona de forma óptima, ya que es necesario completar todos los campos de búsqueda y si no se llena todos los campos de criterio de busqueda el modulo puede cerrarse abruptamente. Las búsquedas solo se pueden realizar por nombres o fechas, no se puede buscar por otros parámetros como por ejemplo por código de pedido o número de sobre u otros.
MÓDULO DE CAJA:
Este subsistema se encarga crear nuevos registros de pedido de lentes por parte del cliente, aquí se copia todos los datos escritos en el sobre rosado del cliente, tales como la fecha del pedido del cliente, fecha de entrega, monto total,monto a cuenta , saldo, nombre del cliente, tipo de montura si es propio o de stock, doctor prescriptor, detalle de los cristales, hora de entrega, código de pedido de los cristales al proveedor, el número de sobre.
Bajo la misma base de datos se guardan estos registros y actualmente solamente funciona en computadoras con sistema operativo Windows 7. 
Este módulo permite visualizar un resumen diario de ventas o pagos cancelados. si anotamos el número de sobre nos devuelve detalles del cliente asociado pero no permite realizar búsquedas bajo otros parámetros. Este módulo no permite generar informes diarios, semanales o mensuales. solo se puede ver el total por día haciendo una especie de búsqueda de la fecha actual, van apareciendo todas las ventas del día y muestra un acumulado en el monto.
Este módulo permite adicionar productos que el cliente ha pedido, ya se cuenta con el módulo de inventarios que se describe más adelante ya que en base al módulo de inventario se puede adicionar productos  como monturas y cristales según disponibilidad y descripción específica, se permite definir el precio final a pesar de tener un precio base definido en el módulo de inventarios. El sistema no avisa la cantidad disponible de la montura , directamente da un mensaje en rojo avisando si ya no hay dicho producto adicionado a la orden de trabajo.
MÓDULO DE INVENTARIO:
Este módulo del sistema se encarga de registrar y controlar las existencias disponibles de la óptica tales como monturas, accesorios, líquidos limpia cristales, estuches, varillas, y otros. Aquí se registran las entradas y salidas y traspasos que pudieran haber.
este módulo permite conocer el saldo actual de cada producto, permite realizar traspasos entre tiendas, permite crear nuevos productos, definir su precio de compra y venta, realizar ingresos por compras o salidas, aunque dichas salidas actualmente sirve para hacer ajustes entre tiendas ya que ambas tiendas trabajan de forma aislada, esto se detalla más adelante. En el caso de las entradas, se toma en cuenta las compras que se hizo de monturas a los proveedores pero el el modulo de caja se registra la venta para restar stock, en el módulo de inventario no se realiza la venta pero el modulo de caja está conectado a este módulo para controlar y restar unidades en caso de ventas. En el caso de los traspasos, no es funcional ya que las tiendas no tienen los datos centralizados y cada tienda gestiona sus datos de forma independiente y aislada y por ende la parte de traspasos no toma efecto en ambas tiendas por eso en caso de haber traspaso de productos entre tiendas , se tiene que hacer de un lado una entrada y del otro lado una salida de forma independiente. Este módulo no permite generar informes y obtener estadísticas del inventario.
SITUACIÓN ACTUAL DE LAS OPERACIONES CON EL SISTEMA
Después de lo descrito mencionaré la situación actual.
La empresa Optalvis actualmente atiende dos tiendas ofreciendo lentes lentes a medida según la prescripción o receta que el cliente trae previa revisión y medición ocular por parte de centros que atienden y realizan la medición correspondiente.
El registro se realiza de forma manual tanto para registrar una nueva orden de trabajo, el registro de la prescripción se realiza en hojas prediseñadas para el registro adecuado de la prescripción. Luego estos mismos registros manuales se las pasa al sistema actual.
Este sistema fue instalado en el año 2000, es un programa de escritorio y fue instalado en cada tienda para las correspondientes tareas y procesos que se requieren. Cada sistema opera de forma independiente, es decir, toda la información que se registra en la tienda central es desconocida por la sucursal y viceversa.
El sistema actual en ambas tiendas funciona en equipos con sistema operativo Windows 7.
Algunas funciones hacen que el sistema se cierre sorpresivamente. Por ejemplo, al realizar búsquedas y no darle todos los parámetros requeridos puede causar que el sistema se cierre presentando un error interno de ejecución. Puede ocurrir que en el módulo de caja , inesperadamente no se registren datos ingresados previamente aunque ha ocurrido muy raras veces. 

El sistema está instalado en una computadora designada para cada tienda y se convierte en el centro donde se guardan los datos de forma local en el disco duro de cada equipo. toda la información se guarda en el equipo de cada tienda. Cada tienda cuenta con conexión de internet, sin embargo el sistema actual no está diseñado para funcionar con internet.
Recientemente la tienda central ha actualizado el equipo, y este nuevo equipo tiende mayores capacidades a nivel hardware y en cuanto al software el sistema operativa que se maneja es Window 10. Se tuvo que volver a instalar el sistema actual e inicialmente todo funcionaba perfecto. pasados un par de meses el sistema actual ha dejado de funcionar completamente debido a las actualizaciones automáticas del sistema operativo. Al abrir el sistema muestra un mensaje que dice que algunos archivos han dejado de tener soporte y ya no son compatibles con el sistema operativo actual y los componentes han dejado de funcionar causando que el sistema actual se cierre después de ese mensaje. Esto ha ocurrido en dos equipos que tenían el sistema operativo 10 y tenían instalado el sistema. En la tienda también cuentan con dos equipos y estos equipos tienen aun funcionando windows 7 y aun el sistema actual funciona con normalidad en la tienda central mientras en en la tienda sucursal aun mantienen funcionando su equipo con windows 7 y también funciona con normalidad. Aunque esto ha causado en la tienda central que se tengan que pausar el proceso de registro de prescripciones ya que esto se realizaba en un equipo de oficina designado para este registro y los equipos de las tiendas donde se realizan la venta solo operan registro de caja y arriba en los equipos que actualizaron se realizaban las tareas de registro de prescripciones y registro y control de inventario. y actualmente aún se lleva a cabo los registro de prescripciones en hojas de papel y las órdenes de trabajo siempre se hicieron en sobres rosas y aun siguen haciendo ese registro y luego lo llevan esos datos al módulo de caja.
